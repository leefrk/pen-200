The Secure flag instructs the browser to only send the cookie over encrypted connections, such as HTTPS. This protects the cookie from being sent in clear text and captured over the network.

The HttpOnly flag instructs the browser to deny JavaScript access to the cookie. If this flag is not set, we can use an XSS payload to steal the cookie.

Craft a WordPress plugin that embeds a web shell:

Exploit the web shell to enumerate the target system:

Once the plugin is active, you should be able to access the web shell through the WordPress interface.
Use it to run commands like whoami, uname -a, ls, etc., to gather information about the system.

A basic web shell lets you run commands, but a reverse shell gives you an interactive shell.
Do you know how to generate a reverse shell payload (e.g., using nc or bash)?
You will need to set up a listener on your Kali machine and modify the web shell to connect back to you.

After gaining a full shell, navigate to /tmp/ and look for the flag file.
Use commands like cat /tmp/flag.txt (or similar) to read the flag.
